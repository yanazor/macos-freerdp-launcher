# macOS FreeRDP Launcher

Небольшое macOS-приложение, которое запускает SDL FreeRDP-сессию к любому доступному RDP-хосту, а пароль получает из macOS Keychain.

## Что умеет

- выглядит как обычное `.app`, хотя Xcode не нужен;
- не хранит RDP-пароль в скрипте;
- получает пароль из Keychain во время запуска;
- включает двусторонний clipboard;
- перенаправляет выбранную папку Mac в удалённую сессию;
- задаёт клавиатурную раскладку;
- отделяет конфигурацию от рабочей логики;
- полностью работает локально, GitHub для установки не нужен.

## Требования

- macOS;
- Homebrew;
- FreeRDP с клиентом `sdl-freerdp`;
- доступный RDP/xRDP-хост.

Установка FreeRDP:

```bash
brew install freerdp
```

## Быстрый старт

```bash
cp config.example config.sh
```

Отредактируй `config.sh`, затем:

```bash
./install.sh
```

RDP-пароль нужно сохранить в macOS Keychain с теми же `KEYCHAIN_ACCOUNT` и `KEYCHAIN_SERVICE`, которые указаны в `config.sh`. Предпочтительно сделать это через приложение «Пароли» или Keychain Access.

Приложение появится здесь:

```text
~/Applications/FreeRDP Launcher.app
```

При включённой опции установщик также создаст ярлыки на рабочем столе.

## Безопасность

В примере используется `/cert:tofu`, а не `/cert:ignore`. Для более строгой схемы нужно установить доверенный сертификат или закрепить ожидаемый сертификат способом, поддерживаемым используемой версией FreeRDP.

Никогда не добавляй в репозиторий:

- пароли;
- приватные ключи;
- внутренние сертификаты с приватным материалом;
- реальные внутренние адреса и имена, если они не предназначены для публикации;
- скриншоты с личными данными.
